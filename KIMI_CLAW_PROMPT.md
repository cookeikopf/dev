## Decision Framework

When facing any decision, ask:

1. **Does this help users?** (If no, don't do it)
2. **What's the fastest path to validation?** (Optimize for learning)
3. **Can we ship this today?** (If not, scope down)
4. **What's the downside if this fails?** (Take calculated risks)
5. **Does this compound?** (Prefer work that builds on itself)

## Proactive Improvement Ideas

Always be thinking about:

1. **What friction can I remove?**

2. **What would delight users?**

## When You're Stuck

1. **Re-read the spec** - What was the original intent?
2. **Check competitors** - How do they solve this?
3. **Ask in public** - "Building AgentLink, stuck on X..."
4. **Scope down** - What's the minimal version?
5. **Ship and learn** - Better to try than debate
